# 开源软件仓库提交行为分析系统
## 项目背景
本项目基于《开源软件基础》课程要求，针对知名Python开源项目`requests`（GitHub: https://github.com/psf/requests.git）的提交历史进行多维度分析，挖掘贡献者行为、提交频率、代码变动等规律，全程使用开源工具链实现。

## 功能模块
1. **仓库数据获取**：通过` gitpython `克隆远程仓库，提取所有提交历史（哈希、作者、时间、提交信息、代码变动量等）。
2. **贡献者分析**：统计Top10贡献者提交次数，可视化贡献者分布。
3. **提交趋势分析**：按周/月统计提交频率，展示项目活跃度变化。
4. **代码变动分析**：统计年度新增/删除代码行数，分析项目迭代节奏。
5. **提交信息词云**：提取提交信息关键词，生成词云展示核心开发方向（如bug修复、功能新增等）。

## 生成文件说明
项目运行后将自动输出以下分析结果文件，可直接用于报告展示或二次分析：

1. **commit_history.csv**：完整的提交历史原始数据，包含所有提取的核心字段；
2. **contributor_top10.png**：Top10 贡献者提交次数可视化图表；
3. **commit_trend_W.png/commit_trend_M.png/commit_trend_ME.png**：按周/月/自定义周期统计的提交趋势图；
4. **code_churn_year.png**：年度代码新增/删除行数统计图表；
5. **commit_message_wordcloud.png**：提交信息关键词词云图。
